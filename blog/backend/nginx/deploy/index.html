    前端項目部署-利用Nginx代理前端項目

    Kaye 3/14/2021 Nginx

    最近完成了一個小型的外包項目,在部署考慮了很多不同的平台,最後決定將項目掛在AWS上,井採用Nginx作為web server。

    # AWS服務

    這個項目是一家連鎖火鍋店的後台監控系統,主要是給運營使用的,對於流量與并發的要求并不高,因此我只選用了一台基礎VPS,服務器環境是ubuntu,開啟服務與SSH連接的教程可以看這裡 (opens new window)。

    # 安裝nginx

    完成SSH連接後,首先要切換為root用戶

    sudo -s

    然後用以下命令安裝Nginx

    sudo apt update
sudo apt install nginx

    # 配置nginx

    若果要啟用防火牆,要留意防火墙有沒有對nginx服務與SSH連接進行封鎖(慘痛教訓:我就是把SSH連接牆掉,導致後面一直連不上。)

    可以用以下指令設置防火牆

    sudo ufw allow 'Nginx HTTP' #允許'Nginx HTTP'服務
sudo ufw status #查詢防火牆情況

    Ubuntu 20.04 上的 Nginx 默認啟用一個服務器塊，配置為/var/www/html,我們直接將項目放到這裡就可以。但如果你想在服務器上託管多個站點，就需要配置額外的Server Blocks。 詳細的配置可以參考這裡 (opens new window)。

    # 測驗服務器

    在安装结束后，Ubuntu 会启动Nginx。 Web Server應該已经在运行了。我們可以查詢服務器的公網IP,并在瀏覽器打開http://your_IP。若果看到 Nginx 的默认页面,就说明服务器运行起来了。

    curl -4 icanhazip.com #返回公網IP

    # 上存文件

    從desktop上存文件到ubuntu可以用scp命令,scp是secure copy的縮寫,它的低層是SSH協議,默認22端口。使用scp傳輸數據時，文件和密碼都是加密的，不會泄漏敏感信息。在首次上存時需要提供密鑰,可以通過-i參數來指定。

    基礎用法為: scp -i (你的密鑰的路徑) (你要傳輸的文件的路徑) (username@ip):(要復制的文件的路徑)

    scp -i key.pem file.zip ubuntu@00.00.000.00:/home/ubuntu

    # 文件操作

    我們上存的是zip檔案,需要把檔案移動到var/www/html路徑下并解壓縮。操作時留意用戶的權限,建議先切換到root。

    # 移動檔案
mv 檔案路徑 目的路徑
# 解壓檔案
unzip 檔案路徑
