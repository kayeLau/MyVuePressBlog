    # LazyMan

    # 問題

    实现一个LazyMan，可以按照以下方式调用:

    LazyMan(“Hank”)输出: Hi! This is Hank!

    LazyMan(“Hank”).sleep(10).eat(“dinner”)输出 Hi! This is Hank! //等待10秒.. Wake up after 10 Eat dinner~

    LazyMan(“Hank”).eat(“dinner”).eat(“supper”)输出 Hi This is Hank! Eat dinner~ Eat supper~

    LazyMan(“Hank”).sleepFirst(5).eat(“supper”)输出 //等待5秒 Wake up after 5 Hi This is Hank! Eat supper

    以此类推。

    # 思路

    1.LazyMan()是以函數的形式調用,而不是以new方式創建(工廠模式)

    2.內部方法可以通過鏈式調用串接(每次調用回傳this)

    3.sleep()方法會阻塞線程,需要等待指定時間才執行後續指令(同步)

    4.sleepFirst()方式會被最先執行

    由此可見,任務的執行次序并非固定的,我們需要一個任務隊列去記錄與控制執行順序,完成調用再執行。

    class LazyMan{
    constructor(name){
        this.name = name;
        this.taskqueue = [];
        this.timer = null;
        this.welcome()
    }
    next(){
        clearTimeout(this.timer);
        this.timer = setTimeout(async()=>{
            for(let i = 0;i < this.taskqueue.length;i++){
                await this.taskqueue[i]();
            }
        })
        return this
    }
    welcome(){
        this.taskqueue.push(()=>{
            console.log(`Hi! This is ${this.name}!`);
        })
        return this.next()
    }
    eat(meal){
        this.taskqueue.push(()=>{
            console.log(`Eat ${meal}~`);
        })
        return this.next()
    }
    sleep(time){
        this.taskqueue.push(()=>
            this.sleepPromise(time)
        )
        return this.next()
    }
    sleepBefore(time){
        this.taskqueue.unshift(()=>
            this.sleepPromise(time)
        )
        return this.next()
    }
    sleepPromise(time){
        return new Promise((resolve,reject)=>{
            setTimeout(()=>{
                console.log(`Wake up after ${time}`);
                resolve();
            },time*1000);
        })
    }
}
    function lazyMan(name){
        return new LazyMan(name)
    }

    即使setTimeout()的延遲時間被設定為0,該方法并不會馬上被執行,異步任務會被推入任務隊列中,直到同步任務都完成才會被執行。利用這個特性,我們可以讓next()方法裡的setTimeout()一直不被執行,直到所有任務都被加到this.taskqueue之中,期間我們可以通過調整數組順序控制任務次序。

    另一個問題是,js裡的異步操作是不會阻塞線程的。而本題需要讓sleep()與sleepBefore()方法阻塞線程,因此在遍歷this.taskqueue任務隊列時需要把異步改成同步,讓一個任務完成之後才執行下一個任務。在這裡可以用到await,await運算子可被用來等待 Promise，它只能在 async function內使用。
